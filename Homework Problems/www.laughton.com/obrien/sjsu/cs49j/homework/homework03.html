3A

Vulcan Poker is a game from Alpha Centari. It is a game of skill not luck. To make the game competitive among players of different skill levels, a score adjustment system has been devised. Each player has an adjustment value ranging from 0 to 40. The better the player, the lower the adjustment value. After a round of play, every player's adjustment value is added to his / her score. The player with the highest final score is the winner.

For this assignment, you will complete the Player class to model a player of Vulcan Poker. There is no game play here, just the player. I supply the tester. Your class will not have a main method.

The Player class has:

  • a constructor that takes a String name and an int adjustment value in that order.
    • That means the class will need instance variables to remember the name and adjustment value. Make them private
  • a method getName() which returns the name of the player
  • a method getAdjustment() that returns the adjustment value
  • a method setAdjustment() which will assign a new adjustment value to this player
    • The method will take a the new value as a adjustment value (stuff in parentheses are parameters)

Provide Javadoc for the constructor and all methods. The Javadoc for the class is given to you

3C

Write a CityBus class. A CityBus moves along a straight route from the start to the end . There are stops every 5 blocks. The track is 50 blocks long. The CityBus moves from one stop to another. It can either move toward the end (+ direction) or towards the start (- direction). Write a class CityBus that models this behavior.

CityBus class has a default constructor that sets the bus at the start moving toward the end (+ direction). You can think of this as a number line with only positive numbers and zero. If the bus is pointing towards the end, moving increases the stop number. If the bus is pointing towards the start, moving decreases the stop number

CityBus has these methods

  • public void move(int numberOfStops) - moves the CityBus the specified number of stops in the current direction. Assume the user is well behaved and never tries to move beyond the start or end of the track
  • public void turn() - reverses the direction of the CityBus. If the bus was moving from start to end (+ direction), after the method executes, the bus will be moving toward the start (- direction)
  • public int distanceToStart() - calculates the distance the bus is from the start in blocks
  • public int distanceToEnd() - calculates the distance the bus is from the end in blocks

Do not use if statements. 

Think about what information needs to be remembered and define instance variables to remember that information.

HINT: You can represent the direction with an integer: +1 if the bus is moving toward the end; -1 if it is moving towards the start. Use that information in the move method to move in the correct direction.

HINT: Remember that each stop is 5 blocks from the previous stop. So if a bus has moved 3 stops along the route, its distance from the start is 15. It will probably be easiest to call the start location stop number 0.

Javadoc for the class might look like this:

/**  
 *   Models a bus traveling on a straight route going either direction  
 */
